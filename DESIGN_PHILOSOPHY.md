# 디자인 철학

## 미션

> **"절제로 드러나는 비트코인의 본질."**

---

## OKR

### O1. 점(dot) 하나로 모든 것을 말한다

- **KR1.** 모든 시각 언어를 점(dot) 하나에서 파생시킨다 — 점의 크기, 밀도, 간격만으로 상태를 표현한다
- **KR2.** 아이콘, 일러스트, 장식적 그래픽을 0으로 만든다
- **KR3.** 하나의 디자인 토큰 체계(컬러, 타이포, 간격)로 모든 컴포넌트를 통일한다

### O2. 색은 의미일 때만 존재한다

- **KR1.** 기본 상태는 모노크롬(흑백)이다 — 색이 등장하는 순간 그것은 시그널이다
- **KR2.** 시그널 색상을 2개로 제한하고, 각 색에 단 하나의 의미만 부여한다
- **KR3.** 색상 없이도 모든 정보의 우선순위를 구분할 수 있다

### O3. 비트코인의 철학을 디자인으로 체현한다

- **KR1.** 희소성 — 화면의 모든 요소는 존재 이유를 증명해야 한다. 설명할 수 없으면 제거한다
- **KR2.** 투명성 — 모든 데이터의 출처와 산출 근거를 한 단계 안에 도달할 수 있다
- **KR3.** 탈중앙 — 하나의 지표가 화면을 지배하지 않는다. 정보의 위계는 시장이 결정한다

### O4. 고요함이 곧 정보가 되게 한다

- **KR1.** 시장이 안정적일수록 화면이 비워진다 — 여백의 양 자체가 시장 상태를 전달한다
- **KR2.** 텍스트 없이 시각만으로 시장 상태를 인지할 수 있다
- **KR3.** 사용자가 대시보드를 열었을 때 "조용하다"고 느끼면, 그것이 곧 "시장이 평온하다"는 정보다

## 핵심 가치

**대상:** 비트코인의 본질을 이해하고자 하는 사람들

### 1. 소음 없는 시그널

크립토 시장은 시끄럽다. 모든 대시보드가 더 많은 차트, 더 많은 색, 더 많은 알림으로 불안을 부추긴다. 우리는 시그널만 남긴다. 색이 나타나면 그것이 곧 의미다. 화면이 고요하면 시장도 고요한 것이다. 해석이 필요 없다.

### 2. 보는 것만으로 아는 경험

텍스트를 읽기 전에, 화면을 여는 순간 시장의 온도를 느낀다. 점의 밀도, 여백의 넓이, 색의 유무 — 눈이 먼저 이해하고, 머리가 따라온다.

### 3. 비트코인 그 자체에 대한 이해

단순히 가격이 오르고 내리는 것을 넘어, 온체인 데이터와 시장 구조를 통해 비트코인이 왜 존재하고, 지금 어떤 상태인지를 읽을 수 있게 한다. 데이터의 출처와 근거는 항상 투명하다.

### 4. 흔들리지 않는 판단

FOMO와 공포를 유발하는 빨강·초록의 난무 대신, 고요한 화면이 냉정한 시선을 유지하게 한다. 시장이 과열될수록 대시보드는 담담하게 사실만 보여준다. 감정이 아닌 본질로 판단하게 한다.

## 디자인 가이드라인

모든 결정은 미션 **"절제로 드러나는 비트코인의 본질"**에서 출발한다. 이 가이드라인에 명시되지 않은 것은 허용되지 않는다.

#### 상태의 이원성

상태는 하나다: 존재 또는 부재.

- 데이터가 있으면 표시된다.
- 데이터가 없으면 비워진다.

로딩, 에러, 성공을 구분하지 않는다. 차이는 오직 시간 지연과 점의 출현 방식으로만 표현된다.

#### 정보의 위계

정보의 중요도는 시스템이 결정하지 않는다.

모든 지표는 동일한 시각적 위계를 가진다. 사용자의 시선 흐름과 시장 상태에 따라 중요도가 자연스럽게 드러나야 한다.

크기, 위치, 색상으로 중요도를 강제하지 않는다. 차이는 오직 점의 밀도와 변화로만 발생한다.

#### UI의 본질

이 시스템에는 컴포넌트가 존재하지 않는다.

모든 UI는 다음 세 가지로 환원된다:

- **점(dot)**
- **텍스트**
- **여백**

카드, 버튼, 배지 등의 개념은 구현 편의를 위한 내부 용어일 뿐, 사용자에게는 인지되지 않아야 한다. 사용자가 "이건 카드다"라고 느끼는 순간 실패다.

#### 텍스트의 역할

텍스트는 상태를 설명하는 수단이 아니라 확인하는 수단이다.

모든 핵심 지표는 텍스트 없이도 상태를 인지할 수 있어야 한다. 텍스트는 그 해석을 보조하는 2차 레이어로만 존재한다.

다음 경우에만 텍스트를 허용한다:

- 수치 확인 (정확한 값)
- 출처 표시
- 사용자의 의도적 조회 (hover, focus)

그 외 상태 전달은 점(dot), 밀도, 여백으로만 표현한다.

### 1. 컬러

#### 원칙

색의 부재가 기본이다. 색이 나타나는 순간, 그것은 시그널이다.

#### 구조 레이어

**모노크롬 (기본 상태)** — 화면의 100%는 이 팔레트로 구성된다.

| 토큰 | 값 | 용도 |
|------|------|------|
| `bg` | `#f5f5f0` | 페이지 배경 |
| `surface` | `#ffffff` | 카드/영역 배경 |
| `text` | `#1a1a1a` | 본문, 데이터 값 |
| `sub` | `#6b7280` | 보조 텍스트, 라벨 |
| `muted` | `#9ca3af` | 비활성, 출처 표기 |
| `border` | `#d1d5db` | 구분선, 카드 보더 |
| `accent` | `#111111` | 강조 (버튼, 헤드라인) |

**시그널 (의미가 존재할 때만)** — 색은 2개만 존재한다.

| 토큰 | 값 | 의미 |
|------|------|------|
| `signal-heat` | `#e53935` | 과열. 주의가 필요한 상태 |
| `signal-cool` | `#1e88e5` | 침체. 시장이 위축된 상태 |

**중립은 색의 부재다.** 별도의 색을 부여하지 않는다. green, yellow은 존재하지 않는다.

#### 적용 규칙

- 한 화면에 시그널 색상은 동시에 최대 2곳까지만 허용한다.
- 시그널 색상은 텍스트, 점(dot), 보더에만 적용한다. 배경색으로 사용하지 않는다.
- 그라데이션을 사용하지 않는다.

---

### 2. 타이포그래피

#### 원칙

하나의 서체, 최소한의 변주. 크기와 굵기만으로 위계를 만든다.

#### 서체

모노스페이스 단일 서체: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace`

#### 스케일

| 토큰 | 크기 | 굵기 | 행간 | 용도 | 허용 개수 |
|------|------|------|------|------|-----------|
| `headline` | 20px | Bold | 1.3 | 페이지/섹션 제목 | 화면당 1개 |
| `body` | 14px | Regular | 1.6 | 설명, 문장 | 제한 없음 |
| `data` | 13px | Bold | 1.0 | 숫자, 지표 값 | 제한 없음 |
| `caption` | 11px | Regular | 1.5 | 보조 정보, 출처, 시간 | 제한 없음 |

#### 적용 규칙

- 이 4단계 외의 크기를 사용하지 않는다.
- 이탤릭을 사용하지 않는다.
- 밑줄을 장식으로 사용하지 않는다.
- 텍스트 색상은 모노크롬 팔레트(text, sub, muted)만 사용한다. 시그널 색상을 텍스트에 적용하는 경우는 데이터 값에 한정한다.
- 자간(letter-spacing)은 caption에만 `0.02em`을 허용하고, 나머지는 기본값을 유지한다.
- 행간은 위 표의 값만 허용한다. data의 행간 1.0은 숫자의 밀도감을 위한 의도적 선택이다.

---

### 3. 간격

#### 원칙

여백은 장식이 아니라 정보다. 빈 공간을 채우려는 충동을 거부한다.

#### 스케일

4px 기반. 다음 값만 허용한다:

`4 · 8 · 12 · 16 · 24 · 32 · 48 · 64`

#### 적용 규칙

| 대상 | 간격 | 비고 |
|------|------|------|
| 컴포넌트 내부 패딩 | 12px — 16px | 절제. 답답하지 않을 최소값 |
| 컴포넌트 간 간격 | 24px — 32px | 호흡. 각 요소가 독립적으로 존재 |
| 섹션 간 간격 | 48px — 64px | 침묵. 구분이 아니라 쉼 |
| 인라인 요소 간격 | 4px — 8px | 밀착. 관련된 정보끼리 |

- 이 스케일에 없는 값(예: 10px, 18px, 26px, 28px)은 사용하지 않는다.
- 간격으로 그룹핑을 표현한다. 배경색으로 그룹핑하지 않는다. 카드의 보더는 그룹핑이 아니라 정보 영역의 경계로만 허용한다.

---

### 4. 컴포넌트

#### 원칙

모든 컴포넌트는 존재 이유를 증명해야 한다. 설명할 수 없으면 제거한다.

#### 4-1. 카드

카드는 정보의 그릇이 아니라 여백의 경계다.

| 속성 | 값 |
|------|------|
| 배경 | `surface` (#ffffff) |
| 보더 | 1px solid `border` |
| 둥근 모서리 | 0 |
| 그림자 | 없음 |
| 호버 | 보더 색상 `accent` 전환 (200ms) |
| 내부 패딩 | 16px |

- 카드 안에 카드를 중첩하지 않는다.
- 여백만으로 영역이 구분되면 보더도 제거한다.

#### 4-2. 버튼

행동을 유도하지 않는다. 선택지를 조용히 제시한다.

**Primary**

| 속성 | 값 |
|------|------|
| 배경 | `accent` (#111111) |
| 텍스트 | `surface` (#ffffff) |
| 크기 | caption (11px) |
| 패딩 | 8px 16px |
| 둥근 모서리 | 0 |
| 호버 | opacity 0.85 |
| 애니메이션 | 없음 |

**Ghost**

| 속성 | 값 |
|------|------|
| 배경 | 없음 |
| 텍스트 | `sub` (#6b7280) |
| 호버 | 텍스트 색상 `text`로 전환 |
| 보더 | 없음 |
| 애니메이션 | 없음 |

**Disabled (비활성)**

| 속성 | 값 |
|------|------|
| 배경 | Primary일 경우 `accent` 불투명도 0.3 |
| 텍스트 | `muted` |
| 커서 | `default` |

- 이 2종(+ 비활성 상태) 외의 버튼 변형을 만들지 않는다.
- 버튼에 아이콘을 넣지 않는다.

#### 4-3. 인풋

| 속성 | 값 |
|------|------|
| 보더 | 하단 1px solid `border` |
| 포커스 | 하단 보더 `accent`로 전환 |
| 배경 | 투명 |
| 둥근 모서리 | 0 |
| 플레이스홀더 | `muted` 색상 |

- 상하좌우 전체 보더(box)를 사용하지 않는다.
- 포커스 시 그림자, 글로우 효과를 넣지 않는다.

#### 4-4. 배지

| 상태 | 배경 | 텍스트 |
|------|------|------|
| 기본 | 없음, 보더 1px `border` | `sub` |
| 과열 | 없음, 보더 1px `signal-heat` | `signal-heat` |
| 침체 | 없음, 보더 1px `signal-cool` | `signal-cool` |

- 배경색을 채우지 않는다. 보더와 텍스트 색상만으로 표현한다.
- 둥근 모서리 0.

### 5. 레이아웃

#### 원칙

시선은 하나의 방향으로만 흐른다. 강제하지 않는다.

#### 구조

| 속성 | 값 |
|------|------|
| 최대 너비 | 1440px |
| 기본 그리드 | 2컬럼: 사이드(280px) + 메인(유동) |
| 카드 그리드 | 메인 영역 내 단일 컬럼 또는 2컬럼 |
| 카드 간 간격 | 24px |
| 섹션 간 간격 | 48px |

#### 시선 흐름 규칙

- 좌→우, 상→하 단방향. 시선이 되돌아가는 구조를 만들지 않는다.
- 한 섹션에 카드 최대 4개. 초과 시 접기(collapse)로 숨긴다.
- 가장 중요한 정보가 가장 위에, 가장 크게가 아니라 가장 먼저 보이도록 배치한다.

#### 동적 여백 (O4 구현)

시장 상태에 따라 화면의 밀도가 변한다.

| 시장 상태 | 화면 반응 |
|-----------|-----------|
| 안정 (중립) | 보조 카드 접힘, 여백 확대, 핵심 지표만 노출 |
| 변동 (과열/침체) | 관련 카드 펼침, 시그널 색상 등장, 밀도 증가 |

### 6. 모션

#### 원칙

고요함이 기본이다. 움직임은 데이터가 말할 때만 허용된다.

#### 허용되는 모션

| 유형 | 조건 | 지속 시간 |
|------|------|-----------|
| 데이터 전환 | 값이 변경될 때 | 400ms |
| 상태 진입 | 시그널 레벨이 변경될 때 | 600ms |
| 호버 반응 | 사용자 인터랙션 | 200ms |
| 오버레이 진입 | 모달, 툴팁 등 출현 | 200ms |

#### 금지되는 모션

- 로딩 스피너 (점의 출현으로 대체)
- 무한 반복 애니메이션
- 장식적 전환 효과
- 바운스, 스프링 등 과장된 이징

#### 시장 연동

- 시장이 안정적일수록 모션이 줄어든다.
- 모든 모션은 `prefers-reduced-motion`을 존중한다.

### 7. 아이콘 / 그래픽

#### 원칙

점에서 파생되지 않으면 존재하지 않는다.

#### 규칙

- 아이콘을 사용하지 않는다. 점의 크기, 밀도, 배치로 대체한다.
- 일러스트를 사용하지 않는다.
- 로고를 제외한 장식적 그래픽을 사용하지 않는다.
- 점으로 표현할 수 없는 개념은 텍스트로 표현한다.

#### 패턴에 의한 인지

아이콘을 사용하지 않는 대신, 반복되는 패턴과 위치로 의미를 학습시킨다.

동일한 유형의 정보는 항상 동일한 위치, 동일한 점 패턴으로 표현한다. 사용자는 텍스트가 아니라 패턴을 기억하게 된다.

- 각 지표는 고유한 점 패턴(밀도 + 크기 + 배치)을 가진다. 이 패턴은 변경하지 않는다.
- 정보의 위치는 고정한다. 같은 데이터는 항상 같은 자리에 있다.
- 사용자가 패턴만으로 지표를 식별할 수 있을 때, 라벨 텍스트는 보조 역할로 후퇴한다.

### 8. 점(Dot) 시스템

#### 원칙

점은 이 프로젝트의 유일한 시각 원소다. 모든 시각 표현은 점에서 파생된다.

#### 크기 스케일

| 토큰 | 크기 | 용도 |
|------|------|------|
| `dot-min` | 0.4px | 배경 텍스처, 거의 인지되지 않는 존재감 |
| `dot-sm` | 1.0px | 기본 그리드, 정적 패턴 |
| `dot-md` | 2.5px | 강조, 데이터 인디케이터 |
| `dot-lg` | 4.0px | 활성 시그널, 상태 표시 |
| `dot-max` | 6.0px | 극한 상태 (극과열/극침체)에서만 허용 |

- 이 5단계 외의 크기를 사용하지 않는다.
- `dot-max`는 극한 상태에서만 사용하며, 화면당 1개로 제한한다.

#### 밀도 스케일

| 토큰 | 그리드 간격 | 용도 |
|------|------------|------|
| `density-sparse` | 20px | 배경, 안정 상태 |
| `density-normal` | 12px | 기본 카드 패턴 |
| `density-dense` | 8px | 강조, 과열/침체 상태 |

- 밀도는 시장 상태에 연동된다. 안정 → sparse, 변동 → dense.
- 3단계 외의 밀도를 사용하지 않는다.

#### 점 불투명도

| 상태 | 불투명도 |
|------|----------|
| 배경 텍스처 | 0.06 — 0.12 |
| 정적 패턴 | 0.12 — 0.18 |
| 활성 인디케이터 | 0.6 — 1.0 |

#### 적용 규칙

- 점은 항상 원(circle)이다. 다른 형태를 사용하지 않는다.
- 점의 색상은 모노크롬이 기본이다. 시그널 상태에서만 `signal-heat`, `signal-cool`을 허용한다.
- 점 애니메이션은 크기 변화와 불투명도 변화만 허용한다. 위치 이동은 데이터 전환 시에만 허용한다.

---

### 9. 차트 / 데이터 시각화

#### 원칙

차트는 데이터를 장식하지 않는다. 데이터 자체가 형태가 된다.

#### 라인 차트

| 속성 | 값 |
|------|------|
| 선 두께 | 1px |
| 선 색상 | `text` (#1a1a1a) |
| 시그널 상태 선 색상 | `signal-heat` 또는 `signal-cool` |
| 채우기 | 없음 |
| 그리드 라인 | 수평만, 1px dotted `border` |
| 축 라벨 | caption 스케일, `muted` 색상 |

#### 데이터 포인트

- 점(dot)으로 표현한다. 크기는 dot 시스템 스케일을 따른다.
- 호버 시 해당 점만 `dot-md` → `dot-lg`로 확대. 툴팁은 별도 섹션 참고.

#### 히트맵

히트맵은 색으로 정보를 표현하지 않는다. 색이 없어도 데이터 분포를 완전히 이해할 수 있어야 한다.

| 속성 | 값 |
|------|------|
| 셀 형태 | 정사각형, 둥근 모서리 0 |
| 기본 표현 | 셀 내부 점의 밀도와 크기 (모노크롬) |
| 임계값 초과 시 | `signal-heat` 또는 `signal-cool` 점 색상 전환 |
| 셀 간격 | 4px |

- 기본 상태에서 모든 셀은 모노크롬이다. 점의 밀도(sparse → dense)와 크기(dot-sm → dot-lg)로 값의 크기를 표현한다.
- 색은 특정 임계값을 넘었을 때만 등장한다. 색의 등장 자체가 시그널이다.
- 배경색 채우기를 사용하지 않는다. 점만으로 표현한다.

#### 적용 규칙

- 3D 효과, 그림자, 그라데이션 채우기를 사용하지 않는다.
- Y축 그리드는 최대 5개 라인. 그 이상은 소음이다.
- 범례(legend)는 차트 내부에 직접 라벨링한다. 별도 범례 박스를 만들지 않는다.
- 차트 제목은 차트 외부 상단에 caption 스케일로 표기한다.

---

### 10. 테이블

#### 원칙

테이블은 격자가 아니라 정렬된 텍스트다. 선은 최소한으로.

#### 구조

| 속성 | 값 |
|------|------|
| 헤더 배경 | 없음 (투명) |
| 헤더 텍스트 | caption 스케일, `muted` 색상 |
| 행 구분 | 하단 1px solid `border` |
| 열 구분 | 없음 (간격으로 구분) |
| 셀 패딩 | 8px 16px |
| 호버 행 | 배경 `bg` (#f5f5f0) |

#### 적용 규칙

- 세로선을 사용하지 않는다.
- 줄무늬(striped) 배경을 사용하지 않는다.
- 숫자 열은 우측 정렬, 텍스트 열은 좌측 정렬.
- 테이블 외곽 보더를 사용하지 않는다.

---

### 11. 네비게이션

#### 원칙

네비게이션은 존재를 드러내지 않는다. 필요할 때만 인지된다.

#### 사이드바

| 속성 | 값 |
|------|------|
| 너비 | 280px (고정) |
| 배경 | 투명 |
| 항목 패딩 | 8px 12px |
| 비활성 항목 | `sub` 색상, 보더 없음 |
| 활성 항목 | `accent` 배경, `surface` 텍스트 |
| 호버 | 텍스트 `text`로 전환 |
| 항목 간 간격 | 4px |

#### 탭

| 속성 | 값 |
|------|------|
| 비활성 탭 | `muted` 색상 |
| 활성 탭 | `text` 색상, 하단 2px solid `accent` |
| 탭 간 간격 | 24px |
| 전환 | 색상 200ms |

#### 적용 규칙

- 탭은 한 줄로 제한한다. 줄바꿈이 필요하면 탭 수를 줄인다.
- 네비게이션에 아이콘을 사용하지 않는다. 텍스트만.
- 현재 위치 표시는 색상 변화만으로 충분하다. 별도 인디케이터를 추가하지 않는다.

---

### 12. 상태 표현

#### 원칙

상태는 존재 또는 부재뿐이다. 로딩, 에러, 성공을 구분하지 않는다.

#### 존재 (데이터가 있을 때)

- 점이 순서대로 출현(dot-sm → dot-md)하며 데이터가 나타난다.
- 출현 간격: 점당 30ms 스태거, 전체 400ms 이내.
- 콘텐츠 영역의 크기를 미리 확보한다 (레이아웃 시프트 0).

#### 부재 (데이터가 없을 때)

- 영역이 비워진다. 여백만 남는다.
- 데이터가 아직 도착하지 않은 것과 에러로 불러오지 못한 것을 구분하지 않는다. 사용자에게는 동일하게 "비어있음"이다.
- 스피너, 스켈레톤, 에러 메시지, 토스트, 체크마크를 사용하지 않는다.
- 재시도가 필요한 경우, 일정 시간 후 시스템이 자동으로 재요청한다. 사용자에게 행동을 요구하지 않는다.

---

### 13. 모달 / 다이얼로그

#### 원칙

모달은 흐름의 단절이다. 가능하면 사용하지 않는다.

#### 허용 조건

- 되돌릴 수 없는 행동의 확인 (삭제, 설정 변경)에만 허용한다.
- 정보 표시 목적으로 모달을 사용하지 않는다. 인라인으로 해결한다.

#### 구조

| 속성 | 값 |
|------|------|
| 배경 오버레이 | `accent` 불투명도 0.3 |
| 모달 배경 | `surface` |
| 보더 | 1px solid `border` |
| 둥근 모서리 | 0 |
| 그림자 | 없음 |
| 최대 너비 | 400px |
| 내부 패딩 | 24px |
| 진입 애니메이션 | 불투명도 0→1, 200ms |

#### 적용 규칙

- 모달 안에 스크롤을 만들지 않는다. 내용이 길면 모달이 아니라 페이지로 만든다.
- 닫기 버튼은 우상단 텍스트 "닫기" (ghost 스타일). X 아이콘 사용 금지.
- 버튼은 최대 2개 (확인 + 취소). 3개 이상의 선택지를 모달에 담지 않는다.

---

### 14. 툴팁

#### 원칙

툴팁은 속삭임이다. 존재감을 최소화한다.

#### 구조

| 속성 | 값 |
|------|------|
| 배경 | `accent` (#111111) |
| 텍스트 | `surface` (#ffffff) |
| 크기 | caption (11px) |
| 패딩 | 4px 8px |
| 둥근 모서리 | 0 |
| 그림자 | 없음 |
| 화살표 | 없음 |
| 지연 시간 | 300ms (즉시 표시하지 않음) |
| 진입 | 불투명도 0→1, 200ms |

#### 적용 규칙

- 한 줄을 초과하지 않는다. 긴 설명이 필요하면 툴팁이 아니라 컨텍스트 텍스트로.
- 터치 디바이스에서는 툴팁을 표시하지 않는다. 정보를 항상 노출하거나 탭으로 전환한다.
- 툴팁에 링크, 버튼 등 인터랙티브 요소를 넣지 않는다.

---

### 15. 구분선

#### 원칙

구분선은 여백이 부족할 때의 차선책이다. 여백으로 해결할 수 있으면 구분선을 사용하지 않는다.

#### 구조

| 유형 | 스타일 |
|------|--------|
| 기본 | 1px solid `border` |
| 점선 | 1px dotted `border` |

- 2종만 허용한다.
- 세로 구분선을 사용하지 않는다.
- 구분선의 좌우 여백(margin)은 0. 컨테이너 전체 너비를 사용한다.

---

### 16. 숫자 표기법

#### 원칙

숫자는 데이터의 핵심이다. 일관되고 즉시 읽혀야 한다.

#### 규칙

| 유형 | 형식 | 예시 |
|------|------|------|
| 퍼센트 | 소수점 2자리, 부호 항상 표시 | `+3.42%`, `-1.08%` |
| 가격 (KRW) | 천 단위 쉼표, 원 단위 절삭 | `142,350,000` |
| 가격 (USD) | 천 단위 쉼표, 소수점 2자리 | `97,432.50` |
| 배율/비율 | 소수점 2자리 | `1.04`, `0.97` |
| 대형 숫자 | 억/만 단위 축약 | `1.2억`, `3,400만` |
| 타임스탬프 | `HH:mm` 24시간제 | `14:32` |
| 날짜 | `MM.DD` | `03.26` |

#### 적용 규칙

- 소수점 자릿수는 위 표를 따른다. 임의로 늘리거나 줄이지 않는다.
- 양수에는 반드시 `+` 부호를 표시한다. 부호의 유무가 정보다.
- 숫자에 단위를 붙일 때 숫자와 단위 사이에 공백을 넣지 않는다. 예: `3.42%`, `142,350,000원`
- 0은 `0`으로 표기한다. `-` 또는 `N/A`로 대체하지 않는다.
- 데이터 없음은 숫자 자리에 `—`(em dash)를 표기한다.

---

### 17. UI 텍스트 원칙

#### 원칙

말을 아낀다. 한 단어로 충분하면 두 단어를 쓰지 않는다.

#### 어조

- 담담하게. 흥분하지 않고, 겁주지 않는다.
- 존댓말을 사용하지 않는다. 경어체 종결 없이 명사형 또는 평서형으로 끝낸다.
- 예: "설정 완료" (O), "설정이 완료되었습니다" (X)
- 예: "프리미엄 4.2%" (O), "현재 프리미엄이 4.2%입니다!" (X)

#### 금지

- 느낌표를 사용하지 않는다.
- 이모지를 사용하지 않는다.
- "축하", "주의", "경고", "위험" 등 감정을 유도하는 단어를 사용하지 않는다.
- 말줄임표(...)를 사용하지 않는다.

#### 라벨 작성

- 최대 4글자. 예: "김프", "펀딩", "미결제", "도미넌스"
- 영문 약어는 대문자 유지. 예: "BTC", "USDT", "OI"
- 괄호 안 부연 설명을 허용하되, caption 스케일로. 예: `김프 (KRW 프리미엄)`

---

### 18. 이징 커브

#### 원칙

움직임은 자연스럽게 시작하고, 자연스럽게 끝난다. 과장하지 않는다.

#### 허용 이징

| 토큰 | 값 | 용도 |
|------|------|------|
| `ease-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | 데이터 진입, 출현 |
| `ease-in-out` | `cubic-bezier(0.45, 0, 0.55, 1)` | 반복 호흡, 펄스 |
| `ease-fade` | `cubic-bezier(0.4, 0, 0.2, 1)` | 불투명도 전환 |

- 이 3종 외의 이징 커브를 사용하지 않는다.
- `linear`를 사용하지 않는다.
- `ease-in`(가속만)을 사용하지 않는다 — 급정거하는 느낌은 젠에 반한다.
- 스프링, 바운스 이징을 사용하지 않는다.

### 19. z-index 체계

#### 원칙

레이어는 최소한으로. 겹침이 적을수록 좋다.

#### 스케일

| 토큰 | 값 | 용도 |
|------|------|------|
| `z-base` | 0 | 기본 콘텐츠 |
| `z-card` | 1 | 카드 내부 요소 |
| `z-sticky` | 10 | 고정 헤더, 사이드바 |
| `z-overlay` | 20 | 툴팁, 드롭다운 |
| `z-modal` | 30 | 모달 오버레이 |
| `z-ambient` | -1 | 배경 점 패턴 (콘텐츠 뒤) |

- 이 6단계 외의 z-index 값을 사용하지 않는다.
- 임의의 z-index(예: 999, 9999)를 사용하지 않는다.

### 20. 반응형

#### 원칙

작은 화면에서는 더 절제한다. 요소를 축소하는 것이 아니라 덜어낸다.

#### 브레이크포인트

| 토큰 | 너비 | 레이아웃 |
|------|------|----------|
| `mobile` | < 768px | 단일 컬럼, 사이드바 숨김 |
| `tablet` | 768px — 1279px | 단일 컬럼, 축소된 여백 |
| `desktop` | ≥ 1280px | 2컬럼 (사이드 280px + 메인) |

#### mobile (< 768px)

- 사이드바를 숨긴다. 최상단에 탭 네비게이션으로 대체.
- 카드는 단일 컬럼, 전체 너비.
- 카드 간 간격 16px. 섹션 간 간격 32px.
- headline 크기 16px으로 축소.
- 컴포넌트 내부 패딩 12px.
- 차트 높이 최소 200px.

#### tablet (768px — 1279px)

- 사이드바를 숨긴다. 상단 네비게이션.
- 카드 그리드 2컬럼 허용.
- 카드 간 간격 16px. 섹션 간 간격 48px.

#### desktop (≥ 1280px)

- 기본 가이드라인 그대로 적용.

#### 적용 규칙

- 요소를 비례 축소하지 않는다. 작은 화면에서는 보조 정보를 접거나 제거한다.
- 가로 스크롤을 허용하지 않는다. 테이블이 넘치면 열을 줄인다.
- 터치 타겟 최소 크기 44px × 44px.

---

### 21. 접근성

#### 원칙

절제는 배제가 아니다. 모든 사용자가 동등하게 정보에 접근한다.

#### 명암비

| 대상 | 최소 명암비 |
|------|------------|
| 본문 텍스트 (`text` on `bg`) | 4.5:1 이상 (AA) |
| 보조 텍스트 (`sub` on `bg`) | 4.5:1 이상 (AA) |
| 비활성 텍스트 (`muted` on `bg`) | 3:1 이상 (AA Large) |
| 시그널 색상 on `surface` | 4.5:1 이상 (AA) |

#### 포커스 표시

- 키보드 포커스 시 2px solid `accent` 아웃라인.
- 아웃라인 오프셋 2px.
- 포커스 표시를 절대 제거하지 않는다.
- 마우스 클릭 시에는 포커스 아웃라인을 숨긴다 (`:focus-visible`만 사용).

#### 키보드

- 모든 인터랙티브 요소는 Tab으로 접근 가능해야 한다.
- 모달은 Esc로 닫힌다.
- 포커스 순서는 시각적 순서와 일치한다.

#### 모션

- `prefers-reduced-motion: reduce` 시 모든 애니메이션을 비활성화한다.
- 모션이 없어도 모든 정보가 전달되어야 한다.

#### 스크린 리더

- 시그널 색상으로만 전달되는 정보에는 반드시 텍스트 대안을 제공한다.
- 점(dot) 시각화에는 `aria-label`로 상태를 명시한다. 예: `aria-label="시장 상태: 과열"`
- 장식적 요소에는 `aria-hidden="true"`를 부여한다.

---

### 22. 다크모드

#### 원칙

다크모드는 지원하지 않는다.

이 프로젝트의 시각 언어는 "흰 바탕 위의 검은 점"이다. 이것은 스타일 선택이 아니라 철학이다. 점이 드러나려면 밝은 바탕이 필요하다. 배경을 어둡게 만드는 것은 철학의 전제를 뒤집는 것이다.

단, 시스템 설정에 관계없이 항상 라이트 모드를 유지하도록 명시적으로 설정한다.

---

### 23. 링크

#### 원칙

링크는 텍스트의 일부다. 시각적으로 튀지 않는다.

#### 구조

| 속성 | 값 |
|------|------|
| 색상 | `text` (#1a1a1a) |
| 밑줄 | 1px solid `border`, underline-offset 2px |
| 호버 | 밑줄 색상 `accent`로 전환 |
| 방문 후 | 변화 없음 |

- 시그널 색상을 링크에 사용하지 않는다.
- 외부 링크 표시(아이콘, 화살표)를 사용하지 않는다.

---

### 24. 폼 요소 확장

#### 라벨

| 속성 | 값 |
|------|------|
| 위치 | 인풋 상단 |
| 크기 | caption (11px) |
| 색상 | `sub` |
| 간격 | 라벨과 인풋 사이 4px |

#### 인풋 에러 상태

- 하단 보더 색상 `signal-heat`로 전환.
- 에러 메시지: 인풋 하단에 caption 스케일, `signal-heat` 색상, 한 줄.
- 에러 메시지는 사실만 전달한다. 예: "숫자만 입력" (O), "올바른 값을 입력해주세요" (X)

#### 텍스트에어리어

| 속성 | 값 |
|------|------|
| 보더 | 하단 1px solid `border` |
| 포커스 | 하단 보더 `accent` |
| 배경 | 투명 |
| 리사이즈 | 세로만 허용 |
| 최소 높이 | 80px |

#### 셀렉트 (드롭다운)

| 속성 | 값 |
|------|------|
| 트리거 | 인풋과 동일한 스타일 (하단 보더 1px) |
| 드롭다운 배경 | `surface` |
| 드롭다운 보더 | 1px solid `border` |
| 항목 호버 | 배경 `bg` |
| 항목 패딩 | 8px 16px |
| 그림자 | 없음 |
| 둥근 모서리 | 0 |

- 화살표 아이콘 대신 `▾` 텍스트 문자를 사용한다.

#### 체크박스

| 속성 | 값 |
|------|------|
| 크기 | 16px × 16px |
| 보더 | 1px solid `border` |
| 체크 표시 | `accent` 배경 채움 (아이콘 없이 면 채움) |
| 둥근 모서리 | 0 |

- 체크 상태는 면 채움으로 표현한다. 체크마크(✓) 아이콘을 사용하지 않는다.

#### 토글

| 속성 | 값 |
|------|------|
| 크기 | 32px × 16px |
| 비활성 | `border` 배경, `surface` 원형 핸들 |
| 활성 | `accent` 배경, `surface` 원형 핸들 |
| 둥근 모서리 | 핸들만 원형. 트랙은 0 |
| 전환 | 200ms |

- 라디오 버튼은 사용하지 않는다. 선택지가 2개면 토글, 3개 이상이면 탭으로 대체한다.

---

### 25. 접기 / 펼치기 (Collapse)

#### 원칙

접힌 상태가 기본이다. 펼침은 사용자의 의지다.

#### 구조

| 속성 | 값 |
|------|------|
| 트리거 | 텍스트 라벨 + `▸` (접힘) / `▾` (펼침) |
| 트리거 크기 | caption (11px), `sub` 색상 |
| 호버 | 텍스트 `text`로 전환 |
| 펼침 애니메이션 | 높이 전환, 200ms, `ease-out` |
| 내부 패딩 | 상단 12px (트리거와의 간격) |

- 아이콘(chevron, plus/minus)을 사용하지 않는다. 텍스트 문자(`▸`, `▾`)만.
- O4 동적 여백: 시장 안정 시 시스템이 자동으로 보조 카드를 접을 수 있다.

---

### 26. 페이지 전환

#### 원칙

전환은 느끼되 의식하지 않는다.

#### 규칙

- 진입: 불투명도 0→1, 400ms, `ease-out`. 위치 이동 없음.
- 퇴장: 불투명도 1→0, 200ms, `ease-fade`. 퇴장은 진입보다 빠르다.
- 슬라이드, 스케일, 회전 전환을 사용하지 않는다.
- 전환 중 이전 페이지와 새 페이지가 동시에 보이지 않는다.

---

### 27. 스크롤

#### 원칙

스크롤은 자연스러운 흐름이다. 방해하지 않는다.

#### 규칙

- 무한 스크롤을 사용하지 않는다. 콘텐츠는 유한하다.
- 페이지네이션이 필요하면 "더 보기" ghost 버튼 1개로. 페이지 번호를 나열하지 않는다.
- 스크롤바: 시스템 기본. 커스텀 스크롤바를 만들지 않는다.
- 스크롤 스냅, 스크롤 재킹을 사용하지 않는다.
- 사이드바는 `position: sticky`. 메인 콘텐츠와 독립적으로 고정.

---

### 28. 알림 / 노티피케이션

#### 원칙

알림은 최소한의 개입이다. 화면을 점유하지 않는다.

#### 인앱 알림

| 속성 | 값 |
|------|------|
| 위치 | 화면 상단 중앙 |
| 배경 | `surface` |
| 보더 | 1px solid `border` |
| 텍스트 | caption 스케일, `text` 색상 |
| 패딩 | 8px 16px |
| 그림자 | 없음 |
| 둥근 모서리 | 0 |
| 자동 소멸 | 3초 후 불투명도 퇴장 (200ms) |

- 한 줄을 초과하지 않는다.
- 색상으로 심각도를 구분하지 않는다. 모든 알림은 모노크롬이다.
- 동시에 1개만 표시한다. 이전 알림은 즉시 사라진다.
- 닫기 버튼을 넣지 않는다. 자동 소멸만.

#### 외부 알림 (텔레그램)

- 텔레그램 알림 설정 UI는 일반 폼 요소로 구성한다.
- 알림 발송 상태는 텍스트로만 표시. 예: "발송 완료", "미연결"
- 시그널 색상을 사용하지 않는다.

---

### 29. 커서 / 선택

#### 커서

| 대상 | 커서 |
|------|------|
| 기본 | `default` |
| 클릭 가능 요소 | `pointer` |
| 인풋 | `text` |
| 비활성 요소 | `default` (not-allowed 금지) |

- `grab`, `zoom-in`, `crosshair` 등 특수 커서를 사용하지 않는다.
- 커스텀 커서 이미지를 사용하지 않는다.

#### 텍스트 선택 (Selection)

| 속성 | 값 |
|------|------|
| 배경 | `accent` 불투명도 0.15 |
| 텍스트 | 변경 없음 |

---

### 금기 목록

이 프로젝트에서 다음은 허용되지 않는다.

- 그라데이션 배경
- 그림자 (box-shadow, text-shadow)
- 둥근 모서리 (border-radius) — 점(dot)과 토글 핸들은 원형 허용
- 아이콘 라이브러리 (Lucide, Heroicons 등)
- 3개 이상의 시그널 색상이 동시에 존재하는 화면
- 사용자가 요청하지 않은 모션
- 느낌표를 사용한 강조 (UI 텍스트)
- 배경색으로 그룹핑
- 중첩된 카드
- 스케일에 없는 간격 값
- 다크모드
- 커스텀 스크롤바
- 무한 스크롤
- 라디오 버튼
- `not-allowed` 커서
- 슬라이드/스케일/회전 페이지 전환
- 동시 2개 이상의 알림 표시
