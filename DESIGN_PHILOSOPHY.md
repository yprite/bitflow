# 디자인 철학

## 미션

> **"절제로 드러나는 비트코인의 본질."**

---

## OKR

### O1. 점(dot) 하나로 모든 것을 말한다

- **KR1.** 모든 시각 언어를 점(dot) 하나에서 파생시킨다 — 점의 크기, 밀도, 간격만으로 상태를 표현한다
- **KR2.** 아이콘, 일러스트, 장식적 그래픽을 0으로 만든다
- **KR3.** 하나의 디자인 토큰 체계(컬러, 타이포, 간격)로 모든 컴포넌트를 통일한다

### O2. 색은 의미일 때만 존재한다

- **KR1.** 기본 상태는 모노크롬(흑백)이다 — 색이 등장하는 순간 그것은 시그널이다
- **KR2.** 사용되는 색상을 5개 이하로 제한하고, 각 색에 단 하나의 의미만 부여한다
- **KR3.** 색상 없이도 모든 정보의 우선순위를 구분할 수 있다

### O3. 비트코인의 철학을 디자인으로 체현한다

- **KR1.** 희소성 — 화면의 모든 요소는 존재 이유를 증명해야 한다. 설명할 수 없으면 제거한다
- **KR2.** 투명성 — 모든 데이터의 출처와 산출 근거를 한 단계 안에 도달할 수 있다
- **KR3.** 탈중앙 — 하나의 지표가 화면을 지배하지 않는다. 정보의 위계는 시장이 결정한다

### O4. 고요함이 곧 정보가 되게 한다

- **KR1.** 시장이 안정적일수록 화면이 비워진다 — 여백의 양 자체가 시장 상태를 전달한다
- **KR2.** 텍스트 없이 시각만으로 시장 상태를 인지할 수 있다
- **KR3.** 사용자가 대시보드를 열었을 때 "조용하다"고 느끼면, 그것이 곧 "시장이 평온하다"는 정보다

## 핵심 가치

**대상:** 비트코인의 본질을 이해하고자 하는 사람들

### 1. 소음 없는 시그널

크립토 시장은 시끄럽다. 모든 대시보드가 더 많은 차트, 더 많은 색, 더 많은 알림으로 불안을 부추긴다. 우리는 시그널만 남긴다. 색이 나타나면 그것이 곧 의미다. 화면이 고요하면 시장도 고요한 것이다. 해석이 필요 없다.

### 2. 보는 것만으로 아는 경험

텍스트를 읽기 전에, 화면을 여는 순간 시장의 온도를 느낀다. 점의 밀도, 여백의 넓이, 색의 유무 — 눈이 먼저 이해하고, 머리가 따라온다.

### 3. 비트코인 그 자체에 대한 이해

단순히 가격이 오르고 내리는 것을 넘어, 온체인 데이터와 시장 구조를 통해 비트코인이 왜 존재하고, 지금 어떤 상태인지를 읽을 수 있게 한다. 데이터의 출처와 근거는 항상 투명하다.

### 4. 흔들리지 않는 판단

FOMO와 공포를 유발하는 빨강·초록의 난무 대신, 고요한 화면이 냉정한 시선을 유지하게 한다. 시장이 과열될수록 대시보드는 담담하게 사실만 보여준다. 감정이 아닌 본질로 판단하게 한다.

## 디자인 가이드라인

모든 결정은 미션 **"절제로 드러나는 비트코인의 본질"**에서 출발한다. 이 가이드라인에 명시되지 않은 것은 허용되지 않는다.

---

### 1. 컬러

#### 원칙

색의 부재가 기본이다. 색이 나타나는 순간, 그것은 시그널이다.

#### 구조 레이어

**모노크롬 (기본 상태)** — 화면의 100%는 이 팔레트로 구성된다.

| 토큰 | 값 | 용도 |
|------|------|------|
| `bg` | `#f5f5f0` | 페이지 배경 |
| `surface` | `#ffffff` | 카드/영역 배경 |
| `text` | `#1a1a1a` | 본문, 데이터 값 |
| `sub` | `#6b7280` | 보조 텍스트, 라벨 |
| `muted` | `#9ca3af` | 비활성, 출처 표기 |
| `border` | `#d1d5db` | 구분선, 카드 보더 |
| `accent` | `#111111` | 강조 (버튼, 헤드라인) |

**시그널 (의미가 존재할 때만)** — 색은 2개만 존재한다.

| 토큰 | 값 | 의미 |
|------|------|------|
| `signal-heat` | `#e53935` | 과열. 주의가 필요한 상태 |
| `signal-cool` | `#1e88e5` | 침체. 시장이 위축된 상태 |

**중립은 색의 부재다.** 별도의 색을 부여하지 않는다. green, yellow은 존재하지 않는다.

#### 적용 규칙

- 한 화면에 시그널 색상은 동시에 최대 2곳까지만 허용한다.
- 시그널 색상은 텍스트, 점(dot), 보더에만 적용한다. 배경색으로 사용하지 않는다.
- 그라데이션을 사용하지 않는다.

---

### 2. 타이포그래피

#### 원칙

하나의 서체, 최소한의 변주. 크기와 굵기만으로 위계를 만든다.

#### 서체

모노스페이스 단일 서체: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace`

#### 스케일

| 토큰 | 크기 | 굵기 | 용도 | 허용 개수 |
|------|------|------|------|-----------|
| `headline` | 20px | Bold | 페이지/섹션 제목 | 화면당 1개 |
| `body` | 14px | Regular | 설명, 문장 | 제한 없음 |
| `data` | 13px | Bold | 숫자, 지표 값 | 제한 없음 |
| `caption` | 11px | Regular | 보조 정보, 출처, 시간 | 제한 없음 |

#### 적용 규칙

- 이 4단계 외의 크기를 사용하지 않는다.
- 이탤릭을 사용하지 않는다.
- 밑줄을 장식으로 사용하지 않는다.
- 텍스트 색상은 모노크롬 팔레트(text, sub, muted)만 사용한다. 시그널 색상을 텍스트에 적용하는 경우는 데이터 값에 한정한다.
- 자간(letter-spacing)은 caption에만 `0.02em`을 허용하고, 나머지는 기본값을 유지한다.

---

### 3. 간격

#### 원칙

여백은 장식이 아니라 정보다. 빈 공간을 채우려는 충동을 거부한다.

#### 스케일

4px 기반. 다음 값만 허용한다:

`4 · 8 · 12 · 16 · 24 · 32 · 48 · 64`

#### 적용 규칙

| 대상 | 간격 | 비고 |
|------|------|------|
| 컴포넌트 내부 패딩 | 12px — 16px | 절제. 답답하지 않을 최소값 |
| 컴포넌트 간 간격 | 24px — 32px | 호흡. 각 요소가 독립적으로 존재 |
| 섹션 간 간격 | 48px — 64px | 침묵. 구분이 아니라 쉼 |
| 인라인 요소 간격 | 4px — 8px | 밀착. 관련된 정보끼리 |

- 이 스케일에 없는 값(예: 10px, 18px, 26px, 28px)은 사용하지 않는다.
- 간격으로 그룹핑을 표현한다. 보더나 배경색으로 그룹핑하지 않는다.

---

### 4. 컴포넌트

#### 원칙

모든 컴포넌트는 존재 이유를 증명해야 한다. 설명할 수 없으면 제거한다.

#### 4-1. 카드

카드는 정보의 그릇이 아니라 여백의 경계다.

| 속성 | 값 |
|------|------|
| 배경 | `surface` (#ffffff) |
| 보더 | 1px solid `border` |
| 둥근 모서리 | 0 |
| 그림자 | 없음 |
| 호버 | 보더 색상 `accent` 전환 (200ms) |
| 내부 패딩 | 16px |

- 카드 안에 카드를 중첩하지 않는다.
- 여백만으로 영역이 구분되면 보더도 제거한다.

#### 4-2. 버튼

행동을 유도하지 않는다. 선택지를 조용히 제시한다.

**Primary**

| 속성 | 값 |
|------|------|
| 배경 | `accent` (#111111) |
| 텍스트 | `surface` (#ffffff) |
| 크기 | caption (11px) |
| 패딩 | 8px 16px |
| 둥근 모서리 | 0 |
| 호버 | opacity 0.85 |
| 애니메이션 | 없음 |

**Ghost**

| 속성 | 값 |
|------|------|
| 배경 | 없음 |
| 텍스트 | `sub` (#6b7280) |
| 호버 | 텍스트 색상 `text`로 전환 |
| 보더 | 없음 |
| 애니메이션 | 없음 |

- 이 2종 외의 버튼 변형을 만들지 않는다.
- 버튼에 아이콘을 넣지 않는다.

#### 4-3. 인풋

| 속성 | 값 |
|------|------|
| 보더 | 하단 1px solid `border` |
| 포커스 | 하단 보더 `accent`로 전환 |
| 배경 | 투명 |
| 둥근 모서리 | 0 |
| 플레이스홀더 | `muted` 색상 |

- 상하좌우 전체 보더(box)를 사용하지 않는다.
- 포커스 시 그림자, 글로우 효과를 넣지 않는다.

#### 4-4. 배지

| 상태 | 배경 | 텍스트 |
|------|------|------|
| 기본 | 없음, 보더 1px `border` | `sub` |
| 과열 | 없음, 보더 1px `signal-heat` | `signal-heat` |
| 침체 | 없음, 보더 1px `signal-cool` | `signal-cool` |

- 배경색을 채우지 않는다. 보더와 텍스트 색상만으로 표현한다.
- 둥근 모서리 0.

---

### 5. 레이아웃

#### 원칙

시선은 하나의 방향으로만 흐른다. 강제하지 않는다.

#### 구조

| 속성 | 값 |
|------|------|
| 최대 너비 | 1440px |
| 기본 그리드 | 2컬럼: 사이드(280px) + 메인(유동) |
| 카드 그리드 | 메인 영역 내 단일 컬럼 또는 2컬럼 |
| 카드 간 간격 | 24px |
| 섹션 간 간격 | 48px |

#### 시선 흐름 규칙

- 좌→우, 상→하 단방향. 시선이 되돌아가는 구조를 만들지 않는다.
- 한 섹션에 카드 최대 4개. 초과 시 접기(collapse)로 숨긴다.
- 가장 중요한 정보가 가장 위에, 가장 크게가 아니라 가장 먼저 보이도록 배치한다.

#### 동적 여백 (O4 구현)

시장 상태에 따라 화면의 밀도가 변한다.

| 시장 상태 | 화면 반응 |
|-----------|-----------|
| 안정 (중립) | 보조 카드 접힘, 여백 확대, 핵심 지표만 노출 |
| 변동 (과열/침체) | 관련 카드 펼침, 시그널 색상 등장, 밀도 증가 |

---

### 6. 모션

#### 원칙

고요함이 기본이다. 움직임은 데이터가 말할 때만 허용된다.

#### 허용되는 모션

| 유형 | 조건 | 지속 시간 |
|------|------|-----------|
| 데이터 전환 | 값이 변경될 때 | 400ms |
| 상태 진입 | 시그널 레벨이 변경될 때 | 600ms |
| 호버 반응 | 사용자 인터랙션 | 200ms |

#### 금지되는 모션

- 로딩 스피너 (점의 출현으로 대체)
- 무한 반복 애니메이션
- 장식적 전환 효과
- 바운스, 스프링 등 과장된 이징

#### 시장 연동

- 시장이 안정적일수록 모션이 줄어든다.
- 모든 모션은 `prefers-reduced-motion`을 존중한다.

---

### 7. 아이콘 / 그래픽

#### 원칙

점에서 파생되지 않으면 존재하지 않는다.

#### 규칙

- 아이콘을 사용하지 않는다. 점의 크기, 밀도, 배치로 대체한다.
- 일러스트를 사용하지 않는다.
- 로고를 제외한 장식적 그래픽을 사용하지 않는다.
- 점으로 표현할 수 없는 개념은 텍스트로 표현한다.

---

### 금기 목록

이 프로젝트에서 다음은 허용되지 않는다.

- 그라데이션 배경
- 그림자 (box-shadow, text-shadow)
- 둥근 모서리 (border-radius)
- 아이콘 라이브러리 (Lucide, Heroicons 등)
- 3개 이상의 색상이 동시에 존재하는 화면
- 사용자가 요청하지 않은 모션
- 느낌표를 사용한 강조 (UI 텍스트)
- 배경색으로 그룹핑
- 중첩된 카드
- 스케일에 없는 간격 값
