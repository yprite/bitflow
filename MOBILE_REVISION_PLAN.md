# 모바일 페이지 수정 계획

> 디자인 철학 문서 기반 현황 분석 및 개선 방향

---

## 1. 현황 진단

### 1-1. 구조 개요

현재 모바일 페이지는 별도 분리 없이 Tailwind 반응형 클래스(`sm:`, `md:`, `lg:`)로 처리한다.
데스크톱 전용 경로(`/desktop`)는 별도 존재하나, 모바일 전용 최적화는 부재하다.

| 영역 | 현재 상태 | 문제 |
|------|-----------|------|
| 헤더/네비게이션 | `flex-wrap` + gap으로 항목 나열 | 항목 4개 이상 시 줄바꿈 발생, 터치 영역 부족 |
| 메인 콘텐츠 영역 | `max-w-3xl mx-auto px-3` | 모바일 좌우 패딩 12px — 철학 기준 최소값이나 카드 내부와 겹침 |
| 카드 컴포넌트 | `dot-card p-4` 단일 패딩 | 모바일에서 내부 여백 과다, 정보 밀도 저하 |
| 푸터 | `flex-wrap` 링크 나열 | 터치 타겟 간격 불충분 |

### 1-2. 디자인 철학 위반 사항

| 위반 항목 | 위치 | 내용 |
|-----------|------|------|
| 컬러 체계 | `tailwind.config.ts` | `dot-green`, `dot-yellow` 색상 존재 — 철학에서 시그널은 `heat`(빨강), `cool`(파랑) 2색만 허용 |
| 둥근 모서리 | `signal-badge.tsx:74` | ScoreGauge에 `rounded-full` 사용 — 철학은 둥근 모서리 0 |
| 에러 상태 UI | `dashboard.tsx:51-64` | 에러 텍스트 + "다시 시도" 버튼 노출 — 철학은 에러/성공 구분 없이 "부재"로 처리 |
| 로딩 상태 | 다수 페이지 | `OrbitalSilence` 로딩 표시 — 철학은 스피너 금지, 점의 출현으로 대체 |
| 모달 사용 | `home-intro-modal.tsx` | 정보 표시 목적 모달 — 철학은 정보 표시 모달 금지, 인라인 해결 |

### 1-3. 모바일 특화 문제

1. **네비게이션 밀집**: 헤더에 4개 링크 + 로고가 한 줄에 배치, 320px 화면에서 터치 오류 유발
2. **터치 타겟**: 네비게이션 링크 패딩 불충분 (최소 44px 권장 미충족)
3. **카드 간 간격**: `space-y-3`(12px) — 철학 기준 컴포넌트 간 간격은 24-32px
4. **텍스트 크기**: `text-[10px]`, `text-[11px]` 다수 — 모바일에서 가독성 한계
5. **툴팁**: 터치 디바이스에서도 hover 기반 툴팁 표시 — 철학은 터치에서 툴팁 금지
6. **스크롤 성능**: 다수 DotAssemblyReveal 래퍼가 모바일 렌더링 부하 증가

## 2. 수정 계획

### Phase 1: 네비게이션 재설계

**목표**: 네비게이션은 존재를 드러내지 않는다 (철학 11장)

#### 1-1. 헤더 간소화

| 항목 | 현재 | 변경 |
|------|------|------|
| 모바일 헤더 | 로고 + 4개 링크 가로 배치 | 로고만 노출, 네비게이션은 하단 고정 탭으로 분리 |
| 데스크톱 헤더 | 동일 | 변경 없음 (`sm:` 이상에서 기존 유지) |

#### 1-2. 모바일 하단 탭 바 도입

```
파일: src/components/mobile-tab-bar.tsx (신규)
```

- `sm:` 이상에서는 `hidden`
- 하단 고정 (`fixed bottom-0`)
- 텍스트만 사용 (아이콘 금지 — 철학 7장)
- 활성 탭: `text-dot-accent`, 하단 2px solid accent
- 비활성 탭: `text-dot-muted`
- 배경: `bg-white/80 backdrop-blur-sm` (기존 헤더와 동일 처리)
- 높이: 48px (터치 타겟 충족)
- safe-area-inset-bottom 대응

#### 1-3. 헤더 수정

```
파일: src/components/site-chrome.tsx
```

- 모바일(`< sm`)에서 네비게이션 링크 숨김
- 헤더 높이 축소: `py-3` → `py-2` (모바일 전용)
- 하단 탭 바와 겹치지 않도록 main에 `pb-16` 추가 (모바일)

---

### Phase 2: 간격 체계 정비

**목표**: 여백은 정보다 (철학 3장)

#### 2-1. 컴포넌트 간 간격 조정

| 대상 | 현재 | 변경 | 근거 |
|------|------|------|------|
| 대시보드 카드 간 | `space-y-3` (12px) | `space-y-6` (24px) | 컴포넌트 간 간격 24-32px |
| 홈 페이지 섹션 | `space-y-4` (16px) | `space-y-6 sm:space-y-8` (24/32px) | 섹션 간 간격 기준 |
| 메인 좌우 패딩 | `px-3 sm:px-4` | `px-4` (16px 통일) | 4px 스케일 준수, 카드 내부 패딩과 동일 |

#### 2-2. 카드 내부 패딩 분화

```
파일: src/app/globals.css (.dot-card 정의)
```

- 모바일: 내부 패딩 `12px` (절제 — 답답하지 않을 최소값)
- 데스크톱(`sm:` 이상): `16px` (기존 유지)

---

### Phase 3: 컬러 체계 정리

**목표**: 색은 의미일 때만 존재한다 (철학 O2)

#### 3-1. tailwind.config.ts 정리

| 토큰 | 조치 |
|------|------|
| `dot-green` (#00c853) | **제거** — 철학에 없는 색상 |
| `dot-yellow` (#f9a825) | **제거** — 철학에 없는 색상 |
| `dot-red` → `dot-signal-heat` | **이름 변경** — 의미 기반 네이밍 |
| `dot-blue` → `dot-signal-cool` | **이름 변경** — 의미 기반 네이밍 |

#### 3-2. 사용처 마이그레이션

- `dot-green`, `dot-yellow` 사용 컴포넌트 탐색 후 `dot-sub` 또는 시그널 색상으로 대체
- `dot-red` → `dot-signal-heat`, `dot-blue` → `dot-signal-cool` 전역 치환

### Phase 4: 상태 표현 통일

**목표**: 상태는 존재 또는 부재뿐이다 (철학 12장)

#### 4-1. 로딩 상태 개선

```
파일: dashboard.tsx, realtime/page.tsx 등
```

- `OrbitalSilence` 로딩 → 점의 순차 출현 애니메이션으로 교체
- 콘텐츠 영역 크기를 미리 확보 (레이아웃 시프트 0)
- 모바일에서 로딩 시 빈 여백만 표시, 데이터 도착 시 점이 순서대로 출현

#### 4-2. 에러 상태 제거

```
파일: dashboard.tsx:51-64
```

- 에러 메시지 + "다시 시도" 버튼 → **빈 영역**으로 대체
- 시스템이 자동 재요청 (일정 간격 후)
- 사용자에게 행동을 요구하지 않는다

#### 4-3. 홈 인트로 모달 인라인화

```
파일: home-intro-modal.tsx → 인라인 섹션으로 전환
```

- 정보 표시 목적 모달 제거
- 동일 내용을 홈 페이지 내 접기(collapse) 가능한 인라인 섹션으로 재배치

---

### Phase 5: 컴포넌트 시각 보정

**목표**: 모든 UI는 점, 텍스트, 여백으로 환원된다 (철학 근본 원칙)

#### 5-1. 둥근 모서리 제거

전역 검색 대상:
- `rounded-full` → 제거 또는 `rounded-none`
- `rounded-lg`, `rounded-md`, `rounded-xl` → 제거
- 카드, 배지, 게이지 등 모든 요소 둥근 모서리 0

#### 5-2. 그림자 제거

- `shadow-*` 클래스 전역 검색 및 제거
- 카드, 버튼, 모달에서 그림자 없음 확인

#### 5-3. 모바일 텍스트 크기 최소값

| 용도 | 최소 크기 | 비고 |
|------|-----------|------|
| caption / 보조 정보 | 11px | 철학 기준 caption 최소값 |
| body | 14px | 기준 유지 |
| data | 13px | 기준 유지 |

- `text-[10px]` 사용처 → `text-[11px]`로 상향 (모바일 가독성)

---

### Phase 6: 터치 최적화

**목표**: 모바일 사용성 확보

#### 6-1. 터치 타겟

| 요소 | 현재 | 변경 |
|------|------|------|
| 네비게이션 링크 | 텍스트 크기만큼 | 최소 `min-h-[44px]` 확보 |
| 버튼 | `py-2 px-4` | `py-3 px-4` (모바일), 기존 유지 (데스크톱) |
| 푸터 링크 | gap 기반 | `py-2` 추가로 터치 영역 확대 |

#### 6-2. 툴팁 → 터치 대응

```
대상: 모든 tooltip 사용 컴포넌트
```

- 터치 디바이스에서 hover 기반 툴팁 비표시 (철학 14장)
- 대안: 정보를 항상 노출하거나, 탭 시 인라인 텍스트 토글

#### 6-3. safe-area 대응

- 이미 `viewport-fit=cover` 설정 있음
- 하단 탭 바에 `pb-[env(safe-area-inset-bottom)]` 적용
- `env-safe` 클래스 활용 확인

---

### Phase 7: 모션 경량화 (모바일)

**목표**: 고요함이 기본이다 (철학 6장)

#### 7-1. 모바일 모션 축소

```
대상: DotAssemblyReveal, DotMorphTransition 등
```

- 모바일에서 `DotAssemblyReveal` 지연(delay) 단축: 현재 70-390ms → 0-150ms
- `prefers-reduced-motion` 미디어 쿼리 준수 확인
- 모바일에서 동시 애니메이션 수 제한 (렌더링 부하 감소)

#### 7-2. 배경 효과 경량화

```
파일: src/components/motion/ambient/AmbientBackground.tsx
```

- 모바일에서 배경 점 패턴 밀도 축소 (`density-sparse` 강제)
- 또는 모바일에서 배경 애니메이션 완전 비활성화

## 5. 검증 기준

각 Phase 완료 시 아래 기준으로 검증한다:

- [ ] 320px 뷰포트에서 가로 스크롤 없음
- [ ] 모든 터치 타겟 44px 이상
- [ ] 시그널 외 색상(green, yellow) 화면에 없음
- [ ] 둥근 모서리 요소 없음 (로고 제외)
- [ ] 그림자 요소 없음
- [ ] 간격이 4px 스케일(4·8·12·16·24·32·48·64)만 사용
- [ ] 에러 메시지, 스피너, 스켈레톤 없음
- [ ] 터치 디바이스에서 hover 툴팁 없음
- [ ] `prefers-reduced-motion` 준수
- [ ] Lighthouse 모바일 Performance 90+ 유지
